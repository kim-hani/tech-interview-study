컴퓨터 구조의 큰 그림

컴퓨터 구조의 큰 그림

컴퓨터 구조 -- 컴퓨터가 이해하는 정보 -- 데이터 -- 명령어 -- 컴퓨터의 핵심 부품 -- CPU -- 메모리와 캐시 메모리 -- 보조기억장치 -- 입출력장치

컴퓨터가 이해하는 정보 컴퓨터는 자바,C++,파이썬과 같은 프로그래밍 언어를 직접 이해하지 못한다. 컴퓨터는 데이터와 명령어만 이해할 수 있다. 데이터는 숫자,문자,이미지와 같은 정적인 정보를 의미한다. 컴퓨터와 주고받는 정보나 컴퓨터에 저장된 정보 자체를 데이터라고 통칭하기도 한다. 데이터는 명령어에 종속적인 정보이며, 명령의 대상이자 , 명령어의 재료이다. 컴퓨터는 기본적으로 0과 1만을 이해할 수 있으므로 데이터와 명령어 또한 0과 1로 이루어져 있다. 컴퓨터의 CPU는 0과 1만으로 이루어진 다양한 데이터를 활용해 명령어를 실행한다.

컴퓨터의 핵심 부품 컴퓨터의 핵심 부품으로는 CPU(중앙처리장치), 메모리(주기억장치) , 캐시 메모리, 보조기억장치,입출력장치가 있다.

CPU 컴퓨터가 이해하는 정보는 데이터와 명령어가 있다. 이러한 정보를 읽어 들이고, 해석하고 , 실행하는 부품이 CPU이다. 사람의 두뇌와 같은 역할을 한다.

  • 산술논리연산장치(ALU,Arithmetic and Logic Unit) : 사칙 연산, 논리 연산과 같은 연산을 수행할 회로로 구성되어 있는 일종의 계산기. CPU가 처리할 명령어를 실직적으로 연산하는 요소
  • 제어장치(CU,Control Unit) : 명령어를 해석해 제어 신호라는 전기 신호를 내보내는 장치. 제어 신호란 부품을 작동시키기 위한 신호를 말한다.
  • 레지스터(register) : CPU 내부의 작은 임시 저장장치로 , 데이터와 명령어를 처리하는 과정의 중간값을 저장한다. CPU내에는 여러 개의 레지스터가 존재하며 , 각기 다른 이름과 역할이 있다.

메인 메모리와 캐시 메모리 CPU가 읽어 들이고, 해석하고 , 실행하는 모든 정보는 어딘가에 저장되어 있어야 하며, 이 정보를 저장하는 장치가 바로 메인 메모리이다. 메인 메모리는 줄여서 메모리라고 부르며 보통 RAM을 지칭한다. 메모리에는 현재 실행 중인 프로그램을 구성하는 데이터와 명령어를 저장하는 부품이다. 프로그램이 실행되기 위해서는 반드시 메모리에 해당 프로그램을 이루는 데이터와 명령어가 적재되어야 한다. 메모리에는 주소라는 개념이 있고, 휘발성이라는 특성이 있다. CPU가 메모리에 빠르게 접근하기 위해서는 정돈되어 있어야 한다. 이때 사용되는 것이 주소이다. 휘발성은 전원이 공급되지 않을 때 저장하고 있는 정보가 지워지는 특성을 의미한다. 메모리는 휘발성 저장장치로 메모리에 저장된 정보는 컴퓨터의 전원이 꺼지면 모두 삭제된다. 캐시 메모리는 CPU가 조금이라도 더 빨리 메모리에 저장된 값에 접근하기 위해 사용하는 저장장치이다. CPU와 메모리 사이에는 반드시 하나 이상의 캐시 메모리가 있으며 CPU내부 혹은 외부에 존재할 수 있다.

보조기억장치 메모리는 휘발성 저장장치로 전원이 꺼지면 저장된 정보가 모두 사라지게 된다. 이를 보조하기 위해 보조기억장치가 사용된다. 보조기억장치는 전원이 꺼져도 저장된 정보가 사라지지 않는 비휘발성 저장장치이다. CD-ROM,하드 디스크 드라이브(HDD),플래시 메모리(SSD,USB)가 이에 해당한다. 메모리가 현재 실행중인프로그램을 저장한다면, 보조기억장치는 보관할프로그램을 저장한다. 어떠한 프로그램이 실행되기 위해서는 반드시 보조기억장치에서 메모리로 해당 프로그램을 복사해야한다.

입출력장치 입출력장치는 컴퓨터 외부에 연결되어 컴퓨터 내부와 정보를 교환하는 장치를 의미한다. 컴퓨터에 어떠한 입력을 할 때 사용하는 장치가 입력장치, 컴퓨터의 정보를 받기 위해 사용하는 장치가 출력장치이다. 마우스,키보드,마이크가 입력장치에 해당하며 스피커,모니터,프린터가 출력장치에 해당한다.

메인 보드와 버스 컴퓨터의 핵심 부품들은 기판에 연결되어 있다. 이 기판을 메인 보드혹은 마더 보드라고 한다. 메인 보드에 연결된 부품들은 각자의 역할을 적절히 수행하기 위해 서로 정보를 주고받는다. 이때 각 컴퓨터 부품들이 정보를 주고받는 통로를 버스라고 한다. 버스의 종류는 다양하지만, 핵심 부품들을 연결하는 시스템 버스가 가장 중요하다.

컴퓨터가 이해하는 정보 CPU는 0과 1만을 이해할 수 있다. 여기서 0과 1을 나타내는 가장 작은 정보의 단위를 비트라고 하며 1비트는 0또는 1,2개의 정보를 표현할 수 있다. 이에 따라 N비트는 2^N개의 정보를 표현할 수 있다. 프로그램의 크기는 바이트단위로 표현되며 1바이트는 8비트이다. // 표로 작성 구분 비트 1byte 8 비트 1kB 1000바이트 1MB 1000킬로바이트 1GB 1000메가바이트 1TB 1000기가바이트 이러한 단위는 프로그램의 크기를 나타낼 때 사용하는 정보 단위이며 CPU관점에서의 정보 단위로는 워드가 있다. 워드란 CPU가 한 번에 처리할 수 있는 데이터의 크기를 의미한다. CPU는 워드 단위로 프로그램을 읽어 들이고 처리한다. 워드의 크기는 보통 32비트 혹은 64비트이다.

데이터 - 0과 1로 숫자 표현하기 0과 1만을 이해할 수 있는 CPU는 컴퓨터 내부에서 2진법을 사용해 2이상 0이하의 수를 이해한다. 10진법은 숫자 9를 넘어가는 시점에 자리올림하고 16진법은 숫자 15를 넘어가는 시점에 자리올림을 한다.

데이터 - 0과 1로 문자 표현하기 컴퓨터가 이해할 수 있는 문자들의 집합은 문자 집합이라고 한다. 문자 집합에 속한 문자를 컴퓨터가 이해하는 0과 1로 이루어진 문자 코드로 변환하는 과정을 문자 인코딩이라고 한다. 반대로 0과 1로 표현된 문자를 사람이 이해하는 문자로 변환하는 과정을 문자 디코딩이라고 한다. 가장 기본적인 문자 집합에는 아스키가 있다.
